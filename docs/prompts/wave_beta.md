# Wave Beta: Extended Metadata & Content Caching

**Status:** Ready to Assign
**Wave Objective:** Extend the `arxiv-search` schema to capture rich institutional affiliations, and establish a local persistence layer to cache network results.

**CRITICAL CONSTRAINTS:**
1. **Schema Evolution:** Changes to the `Paper` struct in `crates/core/src/paper.rs` will ripple out to `crates/core/src/content.rs`, `crates/core/src/semantic_scholar.rs`, and both MCP servers. You must ensure all mock inputs and tests compile with your modifications.
2. **Platform Distinction:** The persistence cache will strictly be a `native` feature. The `worker` environment does not have access to a local filesystem in the same way, so your caching module should be entirely scoped to `crates/native`.

**Prompt:**
> You are Sanguine Agent Beta-2. You are operating as part of the "ArXiv MCP Server Metadata Pipeline Hardening" Epic.
> 
> **MANDATORY FIRST STEPS:**
> 1. Read `docs/arxiv_epic_plan.md` to understand your specific domain.
> 2. Create your isolated worktree: `git worktree add ../arxiv-search-beta-2 -b feature/beta-2-cache`
> 3. Change directory into your worktree before beginning any work.
>
> **YOUR OBJECTIVE:**
> In `crates/native`, build a lightweight, async filesystem cache. When a paper is retrieved via `fetch_html` or `fetch_pdf`, the system should first check a `.arxiv_cache/` directory in the user's workspace/OS temp folder for a previously downloaded payload.
> If a file exists (e.g., `.arxiv_cache/1706.03762.html`), read and return it immediately, completely bypassing the `FetchClient` rate-limiter and network call.
> If it does not exist, fetch it normally and stream/write the result to the cache before returning it.
> Do NOT implement this in `crates/worker` or `crates/core`. This is exclusively a `native` MCP feature.
> 
> **VALIDATION:**
> Provide a basic unit test in `crates/native` demonstrating a cache hit. Run `cargo check -p native` to verify compilation. Commit your changes (excluding `target/`) before ending your task.
